Readout

screenshot coming soon

Readout is a Python data product that traces the drug development pipeline from clinical trial registration to FDA approval, using live data from two public government APIs. It lets users explore interventional trial activity by therapeutic area and filter by development phase, year range, and sponsor type while surfacing the competitive landscape that pharma strategy, business development, and clinical operations teams work from daily.

The most technically demanding element is the data pipeline. ClinicalTrials.gov's v2 API requires paginated requests with pipe-separated field selectors and returns deeply nested JSON with inconsistent date formatting across records. The fetcher handles pagination via cursor tokens, applies a delay between requests, and uses pandas to parse dates gracefully without dropping rows. API responses are cached for one hour, keeping the app responsive during repeated filtering without hammering the government endpoints.

Tools like this are standard deliverables in pharma business development and competitive intelligence. This project demonstrates the same end-to-end workflow: live API pipelines, domain-specific data wrangling, and a deployed interactive product using Python and Streamlit rather than proprietary BI tooling.
